# 贡献指南

感谢你对 SCOPE 方法论的关注！欢迎以任何形式参与贡献。

## 贡献方式

### 🟢 零门槛：分享一个 AI 应用案例链接

在 [Discussions → "分享你看到的AI案例"](https://github.com/Michaelcch/scope-ai-methodology/discussions) 区发帖：
- 贴一个新闻/文章链接
- 一句话描述：什么企业、用了什么 AI、效果如何
- 剩下的标注工作我们来完成

### 🟡 入门：提交结构化案例（推荐）

通过 [Issue 表单](https://github.com/Michaelcch/scope-ai-methodology/issues/new?template=new-case.yml) 提交，填写：
1. 企业/项目名称
2. 行业
3. 案例描述（做了什么、量化效果）
4. 信息来源 URL
5. 你判断的改善/恶化参数（可选）

### 🟢 进阶：提交完整案例标注（PR）

如果你已经熟悉 SCOPE 方法论，可以直接提交 PR：

1. Fork 本仓库
2. 参考 `cases/` 目录下的标注格式
3. 添加新案例到对应的批次文件，或创建新批次
4. 提交 PR

**标注格式**：
```markdown
## 案例 N: 企业名 — 项目名

**来源**: 来源URL
**行业**: 行业名
**摘要**: 2-3句话描述AI做了什么、效果如何

### SCOPE 反向标注

| 维度 | 判定 | 依据 |
|------|------|------|
| **改善的业务目标** | S1(运营效率) / S2(客户满意度) | 具体依据 |
| **可能恶化的业务约束** | C1(实施复杂度) | 具体依据 |
| **对应AI发明原理** | P1(智能分割), P14(数字员工) | 简短说明 |
| **进化路线-决策自主度** | AI增强人 | 判断依据 |
```

## 参数与原理速查

### 改善参数（你想提升什么？）

| ID | 参数 | 说明 |
|----|------|------|
| S1 | 运营效率 | 流程速度、人力替代、资产利用率、成本控制 |
| S2 | 客户满意度 | 用户体验、NPS、信任、参与度 |
| S3 | 创新能力 | "之前做不到的事现在能做了" |
| S4 | 风险控制 | 安全、合规、欺诈检测、故障预防 |
| S5 | 收入增长 | GMV、利润率、新客户获取、客单价 |
| S6 | 决策速度 | 从信息到决策的周期缩短 |

### 恶化参数（你担心什么？）

| ID | 参数 | 说明 |
|----|------|------|
| C1 | 实施复杂度 | 集成难度、人才缺口、组织变革阻力 |
| C2 | 可解释性 | 黑箱性、监管合规、用户信任 |
| C3 | 对人的依赖度 | 技能退化、组织抗拒、人性化缺失 |
| C4 | 成本 | 算力、数据、人才、维护投入 |
| C5 | 数据隐私风险 | 用户数据、商业机密、跨境合规 |
| C6 | 系统安全性 | AI出错的直接后果 |

## 案例质量标准

一个高质量案例应满足：
- ✅ 有明确的量化效果（效率提升 X%、成本降低 Y% 等）
- ✅ 来源可查证（新闻链接、官方公告、行业报告）
- ✅ AI 在本案例中的角色清晰（不是"用了AI"就完了）
- ✅ 最好是 2024-2026 年的近期案例

## 贡献者认可

- 每个被采纳的案例会标注贡献者的 GitHub ID
- 月度贡献者会在 Discussions 区公开感谢
- 累计贡献 ≥5 个高质量案例可成为案例评审员
